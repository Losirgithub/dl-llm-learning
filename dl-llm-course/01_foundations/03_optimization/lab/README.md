# Lab: 四种优化器可视化对比

## 目标
亲手实现 GD、SGD、Momentum、Adam 四种优化器，在同一个 2D 损失曲面上可视化它们的收敛路径，直观理解四者差异。

## 要求
- 手写四种优化器的更新逻辑（不用 torch.optim，那是黑盒）
- 从同一起点出发，画出等高线 + 收敛路径
- 观察并回答本课末尾的分析问题

## 运行环境
- 推荐机器：3070 笔记本（纯 CPU 计算也可，无需 GPU）
- 预估显存：0（本实验不用 GPU）
- 预估时长：跑代码 30 秒，看图分析 5-10 分钟

## 验收标准
- [ ] 代码能跑，生成 `optimizer_paths.png`
- [ ] 图上能清晰看到四条路径颜色不同
- [ ] 能回答：GD 为什么慢？SGD 为什么抖？Momentum 为什么平滑？Adam 为什么最快？

## 提示
- 损失函数用一个"细长山谷"型的二次函数，故意让 x 方向陡、y 方向平——这种"病态"曲面最能暴露优化器差异
- SGD 的"随机"通过给梯度加高斯噪声模拟（真实 SGD 的噪声源自 mini-batch）
- 四种优化器用完全相同的初始点、迭代次数、学习率基准
