# L4.2 Quiz: PEFT / LoRA / QLoRA

先自己答,再对照参考答案。不要只背结论,重点是能把显存和参数流向讲清楚。

## 问题

1. LoRA 为什么叫"低秩适配"? 写出 `ΔW = B A` 里 `r` 的意义。
2. 如果一个线性层原权重是 `W ∈ R^{4096×4096}`,LoRA 取 `r=8`,只对这个层加 LoRA,新增参数量是多少? 相比原矩阵约少多少倍?
3. QLoRA 里的 "Q" 指什么? 它和普通 LoRA 的关键区别是什么?
4. 为什么常把 LoRA 加到 `q_proj`、`v_proj` 上? 如果加到所有 projection 和 MLP 层,通常会发生什么变化?
5. `lora_alpha` 和 `r` 的关系是什么? 为什么常见设置是 `lora_alpha = 2r`?

## 参考答案

1. LoRA 假设任务适配所需的权重变化 `ΔW` 可以用低秩矩阵近似。`r` 是秩(rank),越大表达能力越强,但可训练参数和显存也越多。
2. `A ∈ R^{8×4096}`,`B ∈ R^{4096×8}`,新增参数 `8×4096 + 4096×8 = 65536`。原矩阵参数 `4096×4096 = 16777216`,约少 `256` 倍。
3. `Q` 是 quantized。QLoRA 把冻结基座模型 4-bit 量化加载,反向传播只更新 LoRA adapter;普通 LoRA 通常基座模型仍以 16-bit/bf16/fp16 形式加载。
4. Attention 的 Q/V 直接影响"看什么"和"输出什么信息",常是高收益低成本位置。加到更多模块会提升表达能力,但训练参数、显存、过拟合风险和训练时间都会增加。
5. LoRA 实际增量常带缩放 `alpha/r`。`alpha` 越大,adapter 更新影响越强。`2r` 是常见经验起点,例如 `r=8, alpha=16`。
