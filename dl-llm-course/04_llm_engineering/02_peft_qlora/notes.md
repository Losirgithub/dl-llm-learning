# L4.2 PEFT(LoRA/QLoRA)· 笔记

## 一、为什么需要 PEFT

### 全量微调的问题

微调 = 在预训练模型上继续训练。**全量微调**更新所有参数，要存:

- 模型权重(fp16):7B ~14GB
- 梯度:~14GB
- 优化器状态(Adam 的 m、v):~28GB
- **合计:56GB 显存**

3070(8GB)、5070Ti(16GB)都放不下。

### PEFT 思路

**PEFT(Parameter-Efficient Fine-Tuning)** = 只训练很少参数，冻结其余。LoRA 是绝对主流。

## 二、LoRA 低秩适配

### 原理

冻结原权重 W，学低秩更新 ΔW = A×B:

$$
y = Wx + \Delta W x = Wx + A B x, \quad A \in \mathbb{R}^{d \times r}, B \in \mathbb{R}^{r \times d}
$$

- W(d×d) 冻结不训
- A(d×r)、B(r×d) 训练，r 很小(8/16)
- 参数量从 d² 降到 2dr

**低秩分解是关键**:不是随机选少量权重训，而是把更新分解成两个小矩阵相乘。比如 d=4096, r=8:

- 全量:16,777,216 参数
- LoRA:65,536 参数(少 256 倍)

### 超参数

- **r(rank)**:低秩的秩。r 越大表达越强但参数多。常用 8、16、64
- **alpha**:缩放因子。实际更新 = `alpha/r × A·B`。常用 r 的 2 倍
- **target_modules**:给哪些层加 LoRA。通常 attention 的 q_proj、v_proj

### 推理时 merge

A·B 可算好加到 W 上: `W' = W + A·B`，推理和原模型一样快(无额外开销)。

## 三、QLoRA = 4-bit 量化 + LoRA

### LoRA 的瓶颈

LoRA 省了训练参数(梯度/优化器)，但**模型权重本身还是要完整加载**(7B fp16 要 14GB)，3070 8GB 放不下。

### QLoRA 解法

把冻结的原始权重量化到 4-bit(省显存)，只让 LoRA 的 A、B 用高精度训练:

```text
QLoRA 训练:
  原权重 W: 4-bit 存储(~5GB for 7B)   ← 冻结,不训练
  LoRA 的 A、B: fp16(~0.5GB)          ← 训练
  梯度/优化器: 只针对 A、B(~0.5GB)

  总显存: ~6GB   ← 3070(8GB)能跑!
```

### QLoRA 三个核心创新

1. **4-bit NormalFloat(nf4)**:专门为正态分布权重设计的 4-bit 格式(权重通常服从正态分布，nf4 的量化点按正态分布密集排布)，比普通 4-bit 精度高
2. **双重量化**:连"量化常数"本身都再量化一次，再省显存
3. **分页优化器**:显存不够时用 CPU 内存兜底，防 OOM

光"量化"早就有了(GPTQ、AWQ)。QLoRA 的贡献是这套组合让 **4-bit 量化 + 训练同时可行**，质量不降。

### 三种方式对比

| | 全量微调 | LoRA | QLoRA |
| --- | --- | --- | --- |
| 训练参数 | 100% | 0.3% | 0.3% |
| 权重精度 | fp16 | fp16 | **4-bit** |
| 7B 显存 | 56GB | 15GB | **6GB** |
| 3070 能跑? | ❌ | ❌ | ✅ |
| 质量 | 最好 | 接近 | 接近 |

## 四、实验:QLoRA 微调 Qwen2.5-1.5B

### 任务

固定格式输出:不管输入什么，回答必须以 `【结果】` 开头、`--AI小深` 结尾。这是基线模型默认不会做的，微调效果明显。

### 关键教训:微调要"推翻强先验"必须给足训练量

| | 第一次(失败) | 第二次(成功) |
| --- | --- | --- |
| epoch | 5 | 40 |
| LoRA r | 8 | 16 |
| lr | 2e-4 | 5e-4 |
| loss 终值 | 5.28 | 0.10 |
| token 准确率 | 33% | 96% |
| 效果 | 没学会 | 学会了 |

Instruct 模型有很强的"详细解释"先验(指令微调训练出来的)。**12 条数据 × 5 epoch 推不翻它**，需要 40 epoch + 更大 LoRA 容量 + 更高学习率。

**适用所有微调**:如果微调没效果，先问"训练量够不够推翻模型的先验"。这也是 Stage 6 相关议题--越狱微调(用有害数据破坏对齐)也需要足量训练才能成功。

### 实验结果(成功后)

```text
微调前: 区块链 -> "区块链是一种分布式数据库技术，它通过..."  (长篇大论)
微调后: 区块链 -> "【结果】区块链使用分布式账本技术--AI小深"  (严格遵守格式)
```

## 五、实验方法论:基线对比

**要证明微调有效，必须有操作前后对比**。训练前先跑一遍评估作为基线，训练后再跑同一套评估，对比才说明效果。没有基线的实验没有说服力--可能只是模型本身就会。

## 七、我的误解与纠正

| 我的理解 | 精确修正 |
| --- | --- |
| "LoRA 参数少是因为训练少量权重" | 关键是**低秩分解**:ΔW = A(d×r)×B(r×d)，r 很小，参数从 d² 降到 2dr。"训练少量"是结果，"低秩分解"是手段 |
| "4-bit 量化让小显存也能训练模型" | 4-bit 主要用于**加载/推理**和 **QLoRA 微调**，不是从零训练。是"让小显存也能**跑/微调**大模型" |
| "QLoRA 核心是量化原始权重" | 方向对，但三个具体创新:nf4 格式 + 双重量化 + 分页优化器。光量化早有(GPTQ)，QLoRA 贡献是组合让 4-bit+训练可行 |
| "微调一次就会有效果" | 不一定。要推翻模型强先验必须给足训练量(epoch/lr/r)。第一次 5 epoch 失败，40 epoch 才成功 |
