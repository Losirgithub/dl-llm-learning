# L2.3 CNN 卷积神经网络 · 笔记

> 增量记录版，结课时优化。

## 一、CNN 相比 MLP 的三大改进（递进关系）

三点是不同层次的设计，别混淆：

### 1. 保留 2D 空间结构（数据格式层面）

- **MLP**：输入立刻展平 (28,28) → (784,)，**2D 结构第一步就死**，之后所有层处理一维向量
- **CNN**：数据全程保持 2D，输入 (32,32) → 中间 (16,16) → (8,8)，始终是二维图

### 2. 局部连接 + 参数共享（单层结构层面）

- **MLP**：每个神经元连全部 784 个输入
- **CNN**：每个神经元只看一个 3×3 局部窗口，且同一个滤波器扫全图（参数共享）

参数量对比：

- MLP `Linear(3072, 64)` = 196,672 参数
- CNN `Conv2d(3, 64, k=3)` = 1,792 参数（**110 倍差距**）

### 3. 平移不变性（前两者的自然结果）

滤波器扫全图 → 数字出现在任何位置都能被同一个"数字检测器"识别。

**记忆要点**：保留 2D → 才能局部看 → 局部+共享带来平移不变性。三层递进。

## 二、CNN 核心组件

### `nn.Conv2d(in_channels, out_channels, kernel_size, padding)`

参数量 = $\text{out} \times \text{in} \times k^2 + \text{out}$

例：`Conv2d(3, 64, kernel_size=3)` = $64 \times 3 \times 9 + 64 = 1792$

物理意义：

- `out_channels` 个"特征检测器"
- 每个检测器要看所有 `in_channels` 输入通道，所以有 in 个 $k \times k$ 小矩阵
- 一个检测器参数 = $\text{in} \times k^2$
- 总共 out 个检测器 + out 个 bias

### `nn.MaxPool2d(kernel_size, stride)`

每 $k \times k$ 区域取最大值。作用：

- 尺寸减半（`MaxPool2d(2)`）
- 提供平移不变性
- 扩大感受野

### `nn.BatchNorm2d(num_features)` —— 深度学习最重要技术之一

**在做什么**：对一个 batch 按每个通道分别归一化到 μ=0, σ²=1，再学 γ, β 缩放。

$$
\hat{x} = \frac{x - \mu_{\text{batch}}}{\sqrt{\sigma^2_{\text{batch}} + \epsilon}}, \quad y = \gamma \hat{x} + \beta
$$

**解决什么问题**：**内部协变量偏移**——深层网络每层的输入分布随训练不断变化，像"射箭时靶子在动"。BN 强行把每层输入分布钉死。

**四个好处**：

1. 训练更快（能用更大 lr）
2. 训练更稳（不爆炸不消失）
3. 有轻度正则化（batch 里样本互相影响）
4. 对初始化不敏感

**两种模式（务必区分）**：

- **训练时**：用当前 batch 的 μ, σ² 归一化，同时累积滑动平均 μ_run, σ²_run
- **推理时**：用累积的 μ_run, σ²_run（不再看当前 batch）

**这就是必须 `model.train()` / `model.eval()` 的核心原因**。忘了 `model.eval()`，单张图 batch=1 时 σ²=0 会崩。

**现代替代**：Transformer 时代 batch 太小，用 **LayerNorm**（不依赖 batch，Stage 3 讲）。

### CBR 三件套（现代 CNN 标准模式）

```python
nn.Conv2d(in, out, 3, padding=1),
nn.BatchNorm2d(out),
nn.ReLU(),
```

**顺序固定**：先卷积拿特征，再 BN 稳分布，最后 ReLU 加非线性。所有现代 CNN 都用这个。

## 三、CNN 演进史（建立直觉）

- **LeNet (1998)**：第一个 CNN
- **AlexNet (2012)**：深度学习引爆
- **VGG (2014)**：全 3×3 小卷积堆叠
- **ResNet (2015)**：残差连接，训到 100+ 层，**里程碑**
- **EfficientNet (2019)**：架构搜索
- 现在部分被 Vision Transformer 取代，但 CNN 仍是视觉骨干

## 四、我的误解与纠正 / 补充问答

### 1. "保留 2D 结构" 和 "局部连接" 是一回事吗？

**不是。是两件独立的事**：

- **保留 2D 结构**：数据格式层面（数据是否保持二维）
- **局部连接**：单层结构层面（一个神经元连多少输入）

理论上可以有"数据 2D 但全连接"的层（虽然没人这么用）。CNN 三大优势 = 2D 数据格式 + 局部连接+参数共享 + 平移不变性，**三个递进层次**。

### 2. BatchNorm 到底在解决什么

参见"二、核心组件"的 BN 部分——**内部协变量偏移**。BN 把每层输入分布归一化到标准形式，让深层网络训练稳定。

### 3. CNN 能识别不同大小的图片吗

**卷积部分可以，分类头不行**。

- **卷积部分**：3×3 滤波器只看局部，不管图多大都能扫。同一份权重能用于任意输入尺寸
- **分类头**：`Linear(2048, 256)` 的 2048 是硬编码的（来自 128×4×4）。输入变大后 flatten 后维度也变，Linear 报错

**三种解法**：

1. **全局平均池化 GAP**（现代标配）：flatten 前用 `nn.AdaptiveAvgPool2d(1)`，把 (batch, 128, H, W) 压成 (batch, 128)。ResNet/EfficientNet 都用这招
2. **全卷积网络 FCN**：分类头也用 1×1 卷积代替 Linear
3. **预处理 resize**：把不同大小图缩放到固定尺寸（图像分类的常规做法）

**更深一层**：卷积核学的是"相对模式"，3×3 检测的具体是"多大"的边缘取决于经过多少层池化（感受野）。同一模型在 32×32 和 1024×1024 上学到的东西不同，不能直接复用。

### 4. Conv2d 参数量算错

**错误**：`out × in`（漏了 kernel_size²）

**正确**：`out × in × k² + out`

物理意义：一个滤波器不是一个数，是 k×k 小矩阵。

### 5. 训练优化的是什么

**优化滤波器里的数值**（Conv2d 的 weight 和 bias）。呼应 L2.1 的"变量 = 要优化的参数"。

一开始滤波器是**随机数**（Kaiming 初始化），训练后**变成能识别边缘/角点/形状的具体数值**——这就是"从数据学特征"。

### 6. CNN 能识别极端尺度差异的物体吗（大 3 vs 小 3）

**朴素 CNN 无法很好处理**。核心原因：**固定感受野**。

- 一个 3×3 卷积核学"局部模式"，感受野随层数递增
- 大 3 边缘可能宽 50 像素，小 3 边缘只有 3 像素
- 同一个滤波器不能同时是两种检测器
- 类比：CNN 只有一个固定焦距序列的镜头，没法"变焦"

**业界四大解决方案**：

1. **多尺度训练 + 数据增强**（最实用）：训练时随机 resize 输入，强迫模型学"缩放不变"。`RandomResizedCrop` 是标配。
2. **FPN 特征金字塔**（目标检测标配）：同时用浅层高分辨率特征（小目标）+ 深层低分辨率特征（大目标）。YOLO/Faster R-CNN 都用。
3. **图像金字塔 + 滑动窗口**（经典，现在少用）：把输入缩放多次，每尺度都扫。慢，被 FPN 取代。
4. **Vision Transformer (ViT)**（当前最先进）：抛弃卷积，用自注意力，每个 patch 能看全图任意位置。不受感受野限制。代价：需要海量数据训练。

**实际工程流程**（"1024 大 3 vs 小 3" 场景）：

- Step 1：目标检测模型（含 FPN）找出"数字在哪、多大" → 边界框
- Step 2：把每个框 crop + resize 到 32×32 → 送进分类 CNN

这是**"检测 + 分类"两阶段**方案，工业 OCR 的核心。端到端方案则用 ViT + 大规模预训练（CLIP 等）。

**哲学问题**：人类天生尺度不变，靠眼睛变焦 + 大脑抽象化。CNN 没有这两者，ViT 更接近后者。所有神经网络离人类视觉都还远，这是研究前沿。
