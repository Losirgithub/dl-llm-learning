# L5.1 SFT 指令微调 · 笔记

## 一、SFT 在对齐链路的位置

```text
预训练模型 (L3.3,只会续写)
    ↓ SFT (L5.1)
    ↓ RLHF/DPO (L5.2/5.3)
对齐助手 (ChatGPT/Claude)
```

**SFT = Supervised Fine-Tuning(监督微调)**,也叫指令微调。用"指令-回答"对,教预训练模型听指令回答。

## 二、SFT 解决什么:从"续写"到"听指令"

预训练模型只会续写。给"中国的首都是哪?"可能续写成"这个问题很多人问..."而不回答。

SFT 用指令-回答对监督训练:

```text
{"instruction": "中国的首都是哪?", "output": "中国的首都是北京。"}
```

训练目标和 L3.2 字符级 LM 一样--预测下一个 token。但 loss 只算"答案部分",不算指令部分(指令是条件不是要学的)。

## 三、SFT vs 预训练

| | 预训练 (L3.3) | SFT (L5.1) |
| --- | --- | --- |
| 数据 | 海量无标注文本 | 指令-回答对(人工/生成) |
| 目标 | 续写(自监督) | 听指令回答(监督) |
| 数据量 | 万亿 token | 几万~几百万条 |

SFT 数据量远小,但效果巨大--在预训练能力上微调,教任务而非教语言。

## 四、L4.2 做的就是 SFT

- 数据:`{"input": "苹果", "output": "【结果】苹果是一种水果--AI小深"}` -- 就是指令-回答对
- 方法:QLoRA 微调
- 结果:模型学会听"请解释: XXX"指令按格式回答

L4.2 已实操 SFT,本课只补理论。

## 五、SFT 的局限:为什么还需要 RLHF

1. **模仿数据,不真正懂偏好**:学"数据里这么答",不是"人类偏好这样"
2. **数据质量天花板**:标注员答案有上限,模型超不过
3. **没法学"哪个更好"**:SFT 只学标准答案,学不到"A 比 B 好"这种偏好

**RLHF 解决**:用人类偏好数据(A 比 B 好)训练,模型学"什么是好",而非只模仿标准答案。
